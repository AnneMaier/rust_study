### 학습중 발생한 의문 정리

1. Day 1: Rust 환경 및 기초 문법
Rust의 빌드/패키지 관리 도구는 무엇인가? -> Cargo는 Rust의 빌드 시스템이자 패키지 관리 도구. Python의 pip + Poetry 또는 Node.js의 npm과 같은 역할.

fn main() 함수는 어떤 역할을 하는가? -> fn main() 함수는 Rust 프로그램의 유일한 진입점(Entry Point). 프로그램의 모든 실행은 이 함수에서 시작.

Rust에서 변수 선언 기본 규칙은? -> Rust의 모든 변수는 기본적으로 불변(Immutable). 안정성을 높이는 핵심 철학.

변수를 가변으로 선언하는 방법은? -> 변수의 값을 변경하려면 선언 시 mut 키워드를 명시적으로 붙여야 함. 예: let mut x = 5.

2. Day 2: 소유권(Ownership) 심층 분석
소유권 이동과 복사가 발생하는 조건 -> i32 같은 스택 데이터는 복사(Copy). String 같은 힙 데이터는 소유권 이동(Move).

힙 기반 데이터와 스택 기반 데이터란? -> **스택(Stack)**은 크기가 고정된 데이터(i32, bool)를 저장하는 빠른 메모리. **힙(Heap)**은 크기가 동적으로 변하는 데이터(String, Vec<T>)를 저장하는 유연한 메모리.

String::from 사용 시 데이터가 저장되는 방식은? -> i32 값은 스택에 저장. String의 실제 데이터는 힙에 저장되고, String 변수는 힙 주소(포인터) 정보를 스택에 가짐.

String::from 문법이란? -> String::from("...")은 불변 문자열 리터럴(&str)을 복사하여 힙 메모리를 할당하고 소유권이 있는(Owned) String 객체를 생성하는 함수.

소유권 이동이 발생하는 이유는? -> String과 같은 힙 데이터를 할당할 때 소유권을 이동시켜 원본 변수를 무효화함. 이는 이중 해제(Double Free) 같은 치명적인 메모리 버그를 컴파일 시점에 방지하기 위함.

소유권 개념은 특정 문법에만 적용되는가? -> 아님. 소유권은 String, Vec, HashMap 등 모든 힙 기반 데이터를 다룰 때 적용되는 Rust 언어의 가장 근본적인 규칙.
